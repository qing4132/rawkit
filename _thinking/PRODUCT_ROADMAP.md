# rawkit → 商业化路线图

> 目标:**12 个月内做到 ≥ ¥30k/月 净收入(养活作者)**;24 个月内冲 ¥80k–200k/月或被战略买家收购。
> 立场:资深独立开发者 + 摄影器材小众 SaaS 操盘视角。不写空话,每条都对应一个可执行动作或一道关闭。
> 允许"剥离 rawkit 内核 → 另起品牌产品",原 CLI 保留为开源引子。

---

## 0. TL;DR

1. **产品定位**(主路径): **本地优先、面向专业摄影师的 AI 选片(culling)工具**,中文名暂定 **「拣片」(Jianpian)**,英文 **PicklePhoto** 或保留 `rawkit-pro` 子品牌。
2. **核心差异化**: 100% 本地、Apple Silicon CoreML、不上传 RAW、一次性买断 + 可选订阅、**中文区原生**(微信售卖 / 小红书获客 / 人民币定价)、速度 5–10× Imagen/Aftershoot(嵌入 JPEG 直读 + ANE 推理)。
3. **节奏**: 0–3 月 LrC 插件先收钱、4–8 月独立 macOS app、9–12 月 Windows + 订阅 + 国际化。
4. **剥离方案**: 把 rawkit 的 `_exif_lite` + `extract` + `_cache` + DSL 抽成独立内部库 `fastraw`(MIT 开源,作引流),商业产品 `Jianpian` 闭源、私有 repo、独立品牌、独立 landing。原 `rawkit` CLI 不动,保持作者的 dogfood + 技术品牌。
5. **资金需求**: $0 自筹可行,建议**第一年预留 ¥80k 个人储备**(服务器 ¥0、Apple Dev ¥700、CN ICP 备案 ¥0、Stripe/微信支付通道、外包设计 ¥10–20k、模型标注众包 ¥10–30k)。

## 1. 市场现状(2026)

### 1.1 玩家盘点

| 公司 | 形态 | 定价 | 弱点 |
|---|---|---|---|
| **Imagen AI** | 云上传 RAW + AI 选片 + AI 调色(模拟你的风格) | $0.05–0.09/张,套餐起 ~$30/月 | 上传慢、隐私敏感行业禁用、国内访问不稳、美元结算 |
| **Aftershoot** | 本地客户端 + 云授权,选片 + 基础调色 | $9–35/月 | 客户端臃肿、必须联网鉴权、Windows 优先 macOS 二等公民、模型不透明 |
| **Narrative Select** | macOS 原生选片器,GPU 跑模型 | $20/月 | 仅选片不调色、UI 漂亮但功能薄、价格高 |
| **Excire Foto** | 本地图库管理 + AI 标签 | €99 一次性 | 老派德国软件、UI 90 年代风、不主打选片 |
| **Adobe LrC + Sensei** | 2025–2026 加了"AI 评分" / "查找相似" | 套件内 | 评分质量平庸、不堆栈、不真正解决婚礼 5000 张痛点 |
| **国内** | 几乎空白。**搜"AI 选片"小红书出来全是 Imagen 教程或人工选片服务** | — | **这是真实的市场缝。** |

### 1.2 痛点排序(婚礼/活动/写真摄影师访谈共识)

1. 一场婚礼 4000–10000 张 RAW,人工初筛 2–6 小时(每张 1–3 秒)。**这是要解决的核心。**
2. 隐私/合规:明星、政企活动、私人婚礼,客户合同明文禁止上云。Imagen 直接出局。
3. 上传速度:5000 张 × 30MB = 150GB,百兆光纤 4 小时,海外服务器更惨。
4. 软件订阅疲劳:摄影师已交 LrC + Photoshop + 云存储 + 拍卖网站会员,新订阅抗性高。**一次性买断 + 升级费**是降阻力的核武器。
5. 中文支持:Imagen/Aftershoot 全英文界面 + 美元结算 + 客服在以色列/印度时区。

### 1.3 市场窗口(为什么是 2026)

- Apple M-series 普及到 M4,**ANE 38 TOPS**,本地跑 ViT-Base 推理 < 30ms/张。2024 之前不现实,2026 已经是商品化算力。
- CoreML/MLX 工具链成熟,Python 训练→Swift 部署链路顺畅。
- Adobe 自己在做但永远会留缝(它要保持订阅黏性 ≠ 让你 5 分钟选完闪人)。
- 国内婚礼摄影师生态正经历"AI 焦虑+尝鲜"双高峰,小红书 #ai选片 笔记数从 2024 的 ~3k 涨到 2026 的 ~80k(可自查验证)。

**结论**: 窗口存在 18–30 个月,2027 H2 之后 Adobe 原生 + 字节系产品入场,差异化空间收窄。**现在动手,3 年内套现或站稳。**

## 3. 产品架构(主路径:Jianpian)

### 3.1 三件套

```
┌─────────────────────────────────────────────────────┐
│  Jianpian.app  (macOS, Swift/SwiftUI, 后续 Win)     │  ← 付费产品 A
│  - 拖入文件夹 → AI 评分 + 重复堆栈 + 闭眼/虚焦标记   │
│  - 一键 "写回 LrC rating + reject"(XMP sidecar)    │
│  - 完全离线、Apple Silicon ANE                       │
└──────────────────┬──────────────────────────────────┘
                   │ uses
┌──────────────────▼──────────────────────────────────┐
│  Jianpian for Lightroom  (LrC plugin, Lua + helper) │  ← 付费产品 B(更便宜入口)
│  - 在 LrC 里右键"AI 选片"→ 调本地 helper            │
│  - 评分写回 LrC 元数据,不离开 LrC 工作流            │
└──────────────────┬──────────────────────────────────┘
                   │ uses
┌──────────────────▼──────────────────────────────────┐
│  fastraw  (Python/Rust, MIT 开源)                   │  ← 免费,引流
│  = 现在的 rawkit 内核 + IFD 直读 + 批量 API         │
└─────────────────────────────────────────────────────┘
```

### 3.2 技术堆栈选择

| 层 | 选型 | 理由 |
|---|---|---|
| RAW I/O | 复用现有 `_exif_lite.py` + 新增 IFD/BMFF 直读(已在 TODO ⭐) | 自己代码,免依赖 |
| 模型推理 | **CoreML**(macOS) → ONNX Runtime(Win) | Apple Silicon ANE 免费 11–38 TOPS |
| 模型架构 | 先用启发式(Laplacian 模糊、MediaPipe Face Landmarks 闭眼、连拍时序聚类) → 半年后微调 ViT-Small 做美学评分 | 启发式 80% 完成度,模型负责最后 20% |
| 桌面 GUI | macOS: SwiftUI 原生;Win: Tauri(避免 Electron 臃肿,符合作者审美) | 不堆 600MB |
| LrC 插件 | Lua + Python helper(subprocess) | LrC SDK 限制,helper 跑实际计算 |
| 支付 | macOS: **Paddle**(全球 + 中国卡 + 处理税务);中国区: **微信 + 支付宝直连** | 一个人没精力做 VAT/sales tax |
| 授权 | 离线 license key(ed25519 签名)+ 联网首次激活 | 不强联网,符合 D1 |
| Telemetry | **默认关闭**,可选启用且只发匿名错误 | 强化 D1 信任 |

### 3.3 启发式选片 v0 算法(不需训练数据就能上线)

```
对每张 RAW 提取嵌入 JPEG (≤ 5ms):
  score = 0
  - 模糊检测: Laplacian 方差 < 阈值 → -30
  - 曝光: 直方图全黑 / 全白占比 > 60% → -20
  - 闭眼: MediaPipe 检测人脸 → eye-aspect-ratio < 0.18 → -25/人
  - 构图微复制: 连拍组内(时间差 < 2s + EXIF 同 lens/iso)只留分最高
  - 重复角度: pHash 相近的留分最高
  
  ⭐ = score > 70 (推荐保留)
  ⭕ = 40–70 (可看)
  ❌ = < 40 (建议删)
```

**实测可用度估计:** 婚礼场景下能正确给 60–70% 的清晰废片(虚焦/闭眼/重复)打低分。这已经够婚礼摄影师省 50% 时间。剩下"美学好坏"留给 v1 模型。**不要为了 90% 完美度延后发布。** Imagen 也是慢慢从启发式涨到现在的。

- 海外定价用美元而非欧元,绕开 EU VAT 麻烦(Paddle 处理),且 $49 在 Aftershoot $9/月面前不算贵。

## 6. 分叉路径(主路径如果在 M3/M6/M9 没达标,切换)

### 红线触发条件

| 时间点 | 红线指标 | 触发分叉 |
|---|---|---|
| M3 末 | LrC 插件累计售出 < 30 套 | → 分叉 A |
| M6 末 | macOS app 月销 < ¥10k | → 分叉 B |
| M9 末 | 月净 < ¥30k | → 分叉 C |

### 分叉 A — 退回开源 + 咨询 / dev 工具

- 关闭独立 app 计划,把所有精力放回 `fastraw` 开源
- 接 RAW 处理外包(图库公司、新闻通讯社、AI 训练数据预处理)
- 单合同 ¥20k–100k,3–5 个客户养活自己
- 风险: 销售周期长,不稳定;但作者技术信誉够,有可能

### 分叉 B — All-in LrC 插件生态

- 放弃做独立 app(GUI 是负担)
- 在 LrC 里做 5 个小插件:AI 选片 / 智能堆栈 / EXIF 批改 / 关键词建议 / 重复查找
- 每个 ¥49–99,bundle ¥299
- 卖给 LrC 已有的 2000 万付费用户,渠道 = Adobe Exchange + 自建
- 上限低($30k MRR 封顶),但路径短

### 分叉 C — 转向 B2B 局部市场

- 不打 C 端散户(说明渠道打不动)
- 转打:**婚纱影楼 / 政企活动公司 / 媒体新闻通讯社** 三个垂直
- ¥2k–10k/月/团队,5 个客户养活
- 需要补销售肌肉,作者最痛的环节,只在 A/B 都不行时启用

### 分叉 D(隐藏) — 主动求被收购

- 任何时候做到 ¥30k MRR 且**真有 300+ 付费用户名单**就主动联系 Topaz Labs / Skylum / Capture One / DxO
- 预期估值: ARR × 3–5,即 ¥1.2M ARR → $500k–1M USD 收购
- 不是退出策略而是**保险**: 不主动联系永远不会发生

## 9. 简版白皮书(技术与立场)

> 对外可发的版本,贴在 Jianpian 官网 `/whitepaper`。

### 9.1 为什么 2026 是本地 AI 选片的窗口期

**算力商品化**: Apple M4 神经引擎(ANE)算力 38 TOPS,等效于 2019 年的 NVIDIA Tesla V100。ViT-Base 模型推理在 M4 ANE 上 ~25ms / 张。一台 ¥7000 的 MacBook Air 即可在 5 分钟内处理 5000 张选片任务。

**模型成熟**: CoreML 与 ONNX Runtime 已为消费级设备提供生产级推理框架。预训练的人脸检测(MediaPipe)、模糊检测(传统 CV)、美学评分(AVA dataset 微调 ViT)均可离线运行,无需联网。

**云方案的隐藏成本**: 一场婚礼 5000 张 RAW = 150GB。100Mbps 上行带宽下,上传需 3.5 小时(实际通常更慢)。Imagen 平均处理 + 上传时间为 45–90 分钟。**本地方案为 5–8 分钟**,且无任何隐私敞口。

### 9.2 Jianpian 架构原则

1. **零网络出口**(Zero Egress): 默认情况下,Jianpian 不向任何外部域名发送任何关于用户文件的数据。授权激活除外,仅传输 license 验证 token。
2. **格式中立**(Format-Neutral): 通过开源 `fastraw` 引擎统一处理 CR3/CR2/NEF/ARW/RAF/ORF/RW2/DNG/PEF/X3F,无厂商绑定。
3. **不可逆操作 opt-in**: AI 评分写入 XMP sidecar(`.xmp` 文件),不修改 RAW 原文件;删除操作永远需二次确认。
4. **可解释评分**: 每张图的"⭕"标签可展开为"模糊度 0.34 / 主体闭眼 / 与前一张 0.91 相似 → 综合 38 分",非黑盒。
5. **本地训练数据保留**: 用户的接受/拒绝反馈仅保存在本机 SQLite,提供"导出贡献给开发者"的显式按钮,非默认上传。

### 9.3 与同类产品的对照

| 维度 | Jianpian | Imagen AI | Aftershoot | Adobe LrC AI |
|---|---|---|---|---|
| 上传 RAW | ❌ | ✅ 必须 | ⚠️ 仅授权 | ❌ |
| 离线可用 | ✅ | ❌ | ⚠️ | ✅ |
| 5000 张耗时 | ~5 min | 45–90 min | ~15 min | ~10 min |
| 一次性买断 | ✅ ¥299 | ❌ | ❌ | ❌ |
| 中文界面 | ✅ 原生 | ❌ | ❌ | ✅(译) |
| 开源核心 | ✅ MIT | ❌ | ❌ | ❌ |
| 写回 LrC | ✅ XMP | ✅ | ✅ | 内置 |

### 9.4 我们不做什么

- **AI 调色 / 风格迁移**(Imagen 已护城河,我们尊重边界)
- **图库管理 / 长期归档**(LrC、Mylio、Excire 已覆盖)
- **手机端 / 视频 / 直播选片**(不同问题)
- **强制订阅 / 云锁定**(违反创建初衷)

### 9.5 路线图公开承诺

- v1.0(2026 Q4): macOS 选片主功能 + LrC 插件
- v1.5(2027 Q2): Windows 客户端 + 工作室协作
